Bootstrap disadvantages




  • Reason 1: very comlicatedd css structure
    •
    Due to the many options and configurations of bootstrap the css structure is long an d difficult to undersatnd. To get an element on the desired place you have add a lot of lines to the classname of the element.


  • Reason 2: increased loadingtime of websites
    •
    Because of the many lines of code in the css of bootstrap, your browser has a lot to load. It's because of this reason that the loading time in comparison to other websites is longer.


  • Reason 3: reverse engineering
    •
    When working with bootstrap, I always have a moment where I have a div element and want to move this element. But bootstrap hates it when I try to move things, I dont have the energy to figure out how to move the object in bootstrap code itself. If I am going to do it that way I'll end up with a long code and a short temper. Thats why I mostly will break the bootstrap css code and overide it with my own, with the result being horrible. After an hour or two I just get the urge to pick my laptop and throw it out of a window. So bootstrap fast and easy, nope it isn't. You wil lose more time to correct bootstrap's mistakes then you would creating your own css.


  • Reason 4: taking a hold of your creativity
    •
    Due to the "awesome" bootstrap css stucture, you're website will end up as a bootstrap creation. Almost every website using bootstrap looks the same. it's because of the bootstrap template that you end up using bootstrap's creativity, instead of your own. ofcourse you can chance the bootstrap css, but what use will it have. Taking a template changing the css, and still ending up with your own css. To give you're own twist to bootstrap you will have to change at least 80% of the code you're using, not to mention that the part you're using is 20% of all of bootstap's code.
